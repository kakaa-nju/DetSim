# 深度-状态数指数拟合

## 功能说明

Tracer 现在会实时对搜索深度与状态数的关系进行指数拟合，并在状态监控区显示结果。

## 拟合模型

使用指数模型：`states = a * exp(b * depth)`

拟合方法：对数线性回归
- `ln(states) = ln(a) + b * depth`
- 使用最小二乘法估计参数 `a` 和 `b`

## 显示信息

在状态监控区新增一行显示：

```
DepthFit: N=15 | states=1.23*e^(0.4567*depth) | d+1: 12345 d+5: 678901
```

- `N`: 用于拟合的数据点数量
- `states=...`: 拟合公式
- `d+1`: 预测下一深度的状态数
- `d+5`: 预测5层后的状态数

## 实现细节

### 数据收集
- 每次发现新的最大深度时记录数据点
- 最多保留最近 50 个数据点用于拟合
- 需要至少 3 个点才会显示拟合结果

### 代码位置
- 拟合函数：`scheduler.cpp` - `exponential_fit()`
- 更新函数：`scheduler.cpp` - `update_depth_fit()`
- 显示代码：`status_monitor_thread()` 中 Line 7

### 数据存储
```cpp
static std::vector<std::pair<double, double>> g_depth_data_points;  // (depth, states)
static double g_fit_a;  // 系数
static double g_fit_b;  // 指数
```

## 应用场景

1. **状态空间增长预测**
   - 根据拟合参数预测搜索完整个状态空间需要的状态数
   - 评估当前搜索策略的效率

2. **分支因子分析**
   - `exp(b)` 近似等于平均分支因子
   - 如果 `b ≈ ln(2) ≈ 0.693`，说明每个状态平均产生 2 个新状态

3. **搜索完成时间估计**
   - 结合当前处理速度，估算到达目标深度所需时间

## 示例分析

```
DepthFit: N=20 | states=150.5*e^(0.5802*depth) | d+1: 8912 d+5: 985641
```

分析：
- 初始状态数（depth=0）约为 150
- 平均分支因子 ≈ e^0.5802 ≈ 1.79
- 下一深度预计有 ~8912 个状态
- 5 层后预计有 ~985K 个状态

## 注意事项

1. **数据点少时不显示**：少于 3 个点显示 "collecting data..."
2. **局部拟合**：只使用最近 50 个点，反映当前搜索区域的特性
3. **指数假设**：假设状态数随深度指数增长，某些情况下可能不准确（如分支因子变化）
